Create `schema.md` at project root with the canonical KB entry schema definition. The file must contain:

1. A title section: `# Knowledge Base Entry Schema`

2. Required fields section with exactly these 6 fields, their types, and constraints:

```yaml
concept: string       # Kebab-case slug, unique identifier. Example: "escoliosis-lumbar-diagnostico"
domain: enum          # One of: fisioterapia | ia | finanzas | trading | esoterismo | psicologia | deportes | personal
confidence: float     # Range 0.0-1.0
summary: string       # One sentence describing what this entry asserts
sources: list         # Each item has either note: "notes/slug.md" or url: "https://...", plus date: "YYYY-MM-DD"
last_updated: date    # Format: "YYYY-MM-DD"
```

3. Optional fields section with these 3 groups:

```yaml
contradicts:          # list of {concept: string, detail: string}
extends:              # list of {concept: string}
gaps:                 # list of strings describing unknowns
```

4. Confidence scale table:

| Value | Meaning |
|-------|---------|
| 0.9+  | Multiple independent sources agree, no contradictions |
| 0.7-0.9 | Single high-quality source, or multiple weak sources agreeing |
| 0.5-0.7 | Contested, single influencer source, or unverified claim |
| below 0.5 | Explicitly contradicted or speculative |

5. File naming convention: kebab-case of concept name, no domain prefix, no UUID. Qualifier suffix for multi-aspect concepts. Examples: `escoliosis-lumbar-diagnostico.md`, `pies-planos-tipo.md`

6. Directory structure:
- `kb/concepts/` — all domain knowledge, flat
- `kb/personal/` — personal context entries
- No subdirectories by domain until corpus exceeds ~100 entries

7. A complete example entry showing all required and optional fields in valid YAML frontmatter followed by a markdown body with bullet points.

Then create `kb/INDEX.md` with:
- Title: `# Knowledge Base Index`
- Description: one line explaining this is auto-maintained by the processor
- A table header with columns: Slug | Domain | Summary | Path
- One example row showing the format: `| escoliosis-lumbar-diagnostico | personal | Diagnostico de escoliosis lumbar | kb/personal/escoliosis-lumbar-diagnostico.md |`
- A separator comment: `<!-- Entries below are managed by process.py -->`

Create directories `kb/concepts/` and `kb/personal/` (concepts can be empty with a `.gitkeep`).

import frontmatter, sys, glob
errors = []
