 He estado usando Cloud Code durante más de un año ahora y así he organizado mi fórmula Cloud para obtener el mejor resultado. Empecemos con los agentes. Estos son tus imágenes de AI, cada uno entrenado en un trabajo específico y yo tengo seis de ellos. Por ejemplo, tengo este agente de revisión de código que se asegura de que los problemas de seguridad y de búsqueda de bolsas se resuelvan antes de que se mezcle. Luego tenemos las comandos de slash. Estos son tus cortos de customizaciones donde solo escribes tu prompter y puedes leerlo a cualquier momento sin reescribir nada. Tengo este comando de la fix issue donde solo dejo el número de problema de GitHub y Cloud automáticamente detiene el problema y empieza a arreglarlo. Siguiente, hay hojas. Así que estas son las reglas que siempre debe seguir Cloud. Tengo esta hoja de pre-commit, así que antes de que Cloud pueda cometer cualquier código, se realiza todos estos test primero y si algo falla, el comité se bloquea sin ninguna excepción. Luego hay reglas. Así que en lugar de estallar todo en un archivo masivo, me desplazo a fichas separadas. Tengo este frontend.md para los patrones de componentes y estilización, luego database.md para las encuestas y migración, y Cloud solo usar los relevantes para lo que est trabajando Tambi tienes habilidades Estas son las instrucciones que Cloud sigue cuando reconoce una situaci espec Por ejemplo tengo esta habilidad de dise de frente Y cuando pido a Cloud para construir una página o un sitio web, seguirá estos exactos estándares de diseño, mi paleta de color, mi espacio, todo, de esa particular habilidad. Y Cloud automáticamente elegirá cuando necesita aplicar. Ahora, en la ruta tenemos cloud.mb5. Es básicamente el cerebro de tu proyecto. Y por último, settings.json. Este es tu archivo configurado donde controlas cuáles son las puertas activas, qué puede ejecutar Cloud y qué está completamente fuera de los límites.
